# ClassPet 产品路线图

> 最后更新：2026-07-18（含头脑风暴恢复版）
> 产品哲学：好的行为被看见，而不是差的行为被比较

---

## 竞品对标（classpet.me）

| 维度 | classpet.me | ClassPet（我们） |
|------|-------------|-----------------|
| 宠物表现 | 静态 PNG，换皮 | 有情绪（5档自动）+ CSS 动画 |
| 比较机制 | 显示分数、排名 | **不比分数**，只展示宠物状态 |
| 大屏 | ❌ 无（只有老师手机端） | ✅ 教室大屏花园（不用登录） |
| 家长参与 | ❌ 无 | ✅ 家长扫码看孩子宠物 + 买食物 |
| 上手速度 | 注册后空班，手动建 | 注册即自动演示班，1 分钟上手 |
| 收费 | 收费订阅 | **免费**（老师用） |
| 定位 | "班级管理工具" | "老师的口袋宠物花园" |

## ⚫ 暂不做（明确否决）

- ❌ **排行榜**——违反"不比较"的产品哲学
- ❌ **减分/惩罚**——只做正向激励，不做负向
- ❌ **拍照/上传孩子照片**——隐私风险，幼儿园禁区
- ❌ **强制购买**——先免费铺老师，后面再说
- ❌ **AI行为分析**——噱头，实际意义不大
- ❌ **跨App RPA**——超出技术边界
- ❌ **大型教培机构B端定制**——现阶段不做

---

## 🎨 产品哲学（不变）

1. **好的行为被看见，不是差的行为被比较**
2. **宠物是"活的"**——有情绪、有反应，不是积分计数器
3. **老师完全掌控**——给几分、给不给家长看、大屏开不开，全由老师决定
4. **不打扰家长**——不推送、不通知，想看自己打开
5. **免费铺用户，装扮挣口碑**
6. **上手要快**——注册即用，不要任何配置

## 🏗 技术架构

```
前端：纯 HTML/CSS/JS（无框架）
后端：Node.js + Express + better-sqlite3 (WAL模式)
数据库：SQLite (WAL模式) + JSON文件
认证：JWT (cp_session cookie)
部署：腾讯云轻量服务器 101.34.214.130
守护：systemd (classpet.service + classpet-tunnel)
隧道：Cloudflare Tunnel（待购买域名固化）
端口：应用 3002，隧道代理到 80

五张表：
  teachers / classes / students / scores / shop_purchases

权限模型：
  /teacher → 必须登录（JWT）
  /screen/:code → 不用登录（班级码即钥匙）+ PIN
  /parent/:code → 不用登录（班级码即钥匙）+ PIN
```
